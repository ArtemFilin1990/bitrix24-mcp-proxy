# Contributing Guide

Спасибо за интерес к проекту Bitrix24 MCP Proxy. Чтобы ускорить ревью и не ломать сборку, придерживайтесь следующих правил.

## Workflow
- Разрабатывайте в отдельных ветках от `main`.
- Перед пул-реквестом запускайте `npm run lint`, `npm test` и `npm run build`.
- Описывайте изменения в PR и в `CHANGELOG.md` (раздел `Unreleased`).
- Не коммитьте секреты и токены; используйте `.env` и секреты CI.

## Код-стайл
- TypeScript: ESLint + Prettier (см. конфигурацию в репозитории).
- Без `any`, если это не оправдано и не документировано.
- Для нетривиальной логики добавляйте JSDoc/комментарии.
- Следуйте структуре каталогов (`src/`, `api/`, `.github/`).

## Тесты
- Пишите юнит-тесты для новых функций.
- Мокаем внешние запросы к Bitrix24 через `nock` или `supertest` для HTTP.
- Покрытие не ниже 80% для новых модулей.

## Коммиты
- Сообщения коммитов в стиле `feat: ...`, `fix: ...`, `chore: ...`.
- Один коммит — одна логическая задача.

## Issue Reporting
- Указывайте шаги воспроизведения и ожидаемый результат.
- Прикладывайте логи/скриншоты по возможности.
